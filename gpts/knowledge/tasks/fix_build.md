# Fix Build

Если сборка падает, не надо сразу переписывать проект.

Порядок действий:

1. Запусти реальную команду сборки.
2. Сохрани суть ошибки.
3. Определи слой проблемы:
   зависимость, конфиг, импорт, типы, код, env, файловая структура.
4. Прочитай только связанные файлы.
5. Сделай минимальную правку.
6. Повтори сборку.
7. Если ошибка ушла, сообщи, чем она была вызвана.

Частые причины:

- неправильный импорт;
- отсутствующий файл;
- несовместимая версия зависимости;
- неправильная env-переменная;
- ошибка конфигурации;
- код, который работает локально, но ломается на build step.
