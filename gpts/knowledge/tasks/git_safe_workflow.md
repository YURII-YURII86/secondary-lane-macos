# Git Safe Workflow

Если доступен git, агент должен работать аккуратно.

Порядок:

1. Посмотреть `git status`.
2. Понять, есть ли уже несохранённые изменения.
3. Не трогать чужие несвязанные изменения.
4. После правок посмотреть `git diff`.
5. В отчёте коротко перечислить, что именно изменилось.

Чего не делать без крайней необходимости:

- не откатывать чужие изменения;
- не делать разрушительные команды;
- не переписывать историю;
- не делать массовые механические изменения без проверки.
