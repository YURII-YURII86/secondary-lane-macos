# Server Triage

Если агент работает с удалённым сервером, сначала нужно понять сервер, а не сразу менять конфиги.

Базовый порядок:

1. Определи, где находится нужный сервис.
2. Пойми, чем он запускается:
   systemd, docker, compose, supervisor, screen, tmux или вручную.
3. Проверь статус сервиса.
4. Проверь логи.
5. Найди конфиги и env-файлы.
6. Только после этого меняй то, что относится к проблеме.

Что полезно выяснить:

- пользователь, под которым работает сервис;
- рабочая директория;
- команда запуска;
- какие порты заняты;
- какой reverse proxy стоит сверху;
- как подтверждается здоровье сервиса.
