# Safe Ops

Главное правило: сначала безопасное действие, потом рискованное.

Без причины нельзя:

- удалять файлы;
- массово переписывать каталоги;
- запускать опасные shell-команды;
- менять продовые конфиги без подтверждающих признаков;
- останавливать сервис без плана проверки и возврата.

Перед рискованным шагом:

1. Собери признаки, что шаг действительно нужен.
2. Выбери самый маленький вариант вмешательства.
3. Подумай, как проверить результат сразу после действия.

Если задача не требует destructive действий, избегай их полностью.
