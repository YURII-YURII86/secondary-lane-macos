# Python Projects

Основные признаки:

- есть `pyproject.toml`, `requirements.txt`, `poetry.lock`, `Pipfile`;
- есть папки `app`, `src`, `backend`, `api`;
- есть признаки фреймворков: `fastapi`, `django`, `flask`.

Что проверить сначала:

- где зависимости;
- какая версия Python ожидается;
- какой файл является точкой входа;
- как запускается приложение;
- есть ли миграции, фоновые процессы или воркеры.

Если задача про запуск:

1. Найди команду запуска в `README`, `Makefile`, `Dockerfile` или service-файлах.
2. Проверь env-переменные.
3. Сначала попробуй безопасную диагностику, потом запуск.

Если задача про правку:

1. Определи модуль, который отвечает за проблему.
2. Прочитай только нужные файлы вокруг точки ошибки.
3. После изменения запусти минимальную проверку.
