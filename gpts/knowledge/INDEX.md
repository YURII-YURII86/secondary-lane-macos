# Knowledge Index

Этот набор знаний разделён по смыслу.

Он является частью обязательного контекстного слоя проекта по протоколу
`LLM-Native Context Architecture`, а не просто набором подсказок.

Порядок выбора:

1. Сначала понять тип задачи.
2. Потом понять стек или среду.
3. После этого открыть только нужные файлы.

Разделы:

- `core/agent_rules.md` — общие правила поведения агента.
- `core/diagnostics.md` — как исследовать незнакомый проект или среду.
- `stacks/node.md` — признаки и логика для Node.js проектов.
- `stacks/python.md` — признаки и логика для Python проектов.
- `stacks/react.md` — как читать React-проекты.
- `stacks/nextjs.md` — как читать Next.js проекты.
- `stacks/fastapi.md` — как читать FastAPI проекты.
- `stacks/docker.md` — как понимать Docker и compose.
- `tasks/fix_build.md` — как чинить упавшую сборку.
- `tasks/inspect_deploy.md` — как понимать, как проект разворачивается.
- `tasks/server_triage.md` — как исследовать удалённый сервер.
- `tasks/patch_and_verify.md` — как безопасно вносить изменения и проверять.
- `tasks/git_safe_workflow.md` — как безопасно работать с git.
- `security/safe_ops.md` — чего нельзя делать без причины.
- `reporting/response_format.md` — как коротко и полезно отчитываться.

Связанный глобальный контекст:

- `.ai_context/system_state.json`
- `.ai_context/dependency_graph.md`
- `.ai_context/decision_log.md`
- `.ai_context/active_tasks.json`
