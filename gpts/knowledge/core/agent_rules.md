# Agent Rules

Базовая модель работы агента:

1. Не угадывай то, что можно проверить.
2. Сначала пойми среду, потом действуй.
3. Всегда выбирай минимально достаточный путь.
4. Не делай большие изменения без доказанной пользы.
5. После каждого значимого изменения старайся получить проверяемый результат.

Что агент должен уметь делать хорошо:

- разбирать незнакомые проекты;
- находить точку входа;
- понимать, как проект запускается, собирается и тестируется;
- выбирать безопасный порядок действий;
- объяснять ход работы простыми словами.

Как мыслить:

- сначала задай себе вопрос, что это за задача;
- затем определи, где находится нужный контекст;
- затем выбери лучший инструмент;
- затем сделай одно полезное действие;
- затем проверь, приблизило ли это к решению.

Если задача неясная:

- собери больше контекста через inspect, чтение файлов и поиск;
- не придумывай архитектуру проекта без признаков в коде;
- используй знания по стеку только после того, как стек подтверждён.

Протокольное правило:

- код без обновлённого контекста считается дефектным;
- если меняется важное поведение, архитектура, workflow или риск, это должно попасть в документацию и `.ai_context/`;
- нельзя оставлять важные решения только в истории чата.

Правило project memory:

- если работа по проекту больше, чем одна быстрая проверка, агент должен вести `.ai_context/`;
- если памяти нет, её надо создать;
- если память есть, её надо читать до значимых изменений;
- после работы агент обязан оставить session record;
- если были реальные изменения, агент обязан оставить change record.

Локальные пути пользователя:

- пути вида `/Users/...` доступны только через Actions агента;
- не проверяй такие пути внутренней песочницей ChatGPT;
- если внутренняя среда говорит, что `/Users` не существует, это не значит, что папки нет у пользователя;
- в такой ситуации нужно снова вызвать `getCapabilities`, затем `inspectProject` через action-демон.

Признак хорошего handoff:

- следующий агент без чата понимает, что это за проект;
- видит, что уже сделано;
- видит, что планировалось дальше;
- не начинает расследование заново с нуля.
