# Mac First Start

## RU

### Зачем нужен этот документ

Этот документ нужен для macOS-версии Second Lane.

Его цель простая:

- провести человека от папки с проектом до рабочего GPT с Actions;
- сделать это без догадок и лишней теории;
- показать сразу два пути: полностью вручную и с помощью Codex skill;
- дать очень подробные шаги для новичка.

По этой инструкции ты можешь:

1. подготовить папку проекта на Mac;
2. проверить или поставить Python 3.13;
3. проверить или поставить `ngrok`;
4. создать `authtoken` и домен в `ngrok`;
5. заполнить `.env`;
6. запустить панель `Second Lane`;
7. проверить туннель и `openapi.gpts.yaml`;
8. собрать GPT в ChatGPT;
9. подключить `Actions` и проверить реальный вызов.

---

### Для кого эта инструкция

Эта инструкция подходит, если:

- у тебя Mac;
- ты хочешь запускать проект локально у себя;
- ты хочешь, чтобы ChatGPT мог работать с проектом через GPT Actions;
- ты не хочешь гадать, куда нажимать и что именно копировать.

Она особенно полезна, если ты:

- раньше не настраивал Python-проекты на macOS;
- не работал с `ngrok`;
- не собирал GPT с Actions вручную;
- хочешь потом отдать эту же задачу Codex по готовому skill.

---

### Как читать эту инструкцию

Здесь всё идёт в правильном порядке.

Не перескакивай через шаги.

Правильная последовательность такая:

1. папка проекта;
2. Python 3.13;
3. `ngrok`;
4. `ngrok authtoken`;
5. `ngrok domain`;
6. `.env`;
7. запуск панели;
8. проверка туннеля;
9. проверка `openapi.gpts.yaml`;
10. GPT builder;
11. `Actions`;
12. токен авторизации;
13. тестовый вызов.

Не начинай собирать GPT, пока не появился реальный публичный URL и пока `openapi.gpts.yaml` не обновился.

### Мини-словарь простыми словами

**Python 3.13** — это версия Python, под которую сейчас подтверждён локальный запуск проекта.

**ngrok** — это сервис, который даёт проекту публичный интернет-адрес, чтобы ChatGPT мог достучаться до твоего локального агента.

**`.env`** — это файл с настройками проекта. Именно туда кладутся `AGENT_TOKEN`, `NGROK_DOMAIN` и пути доступа к проектам.

**`AGENT_TOKEN`** — секретный токен для доступа к агенту. Его потом использует GPT в `Actions`.

**`openapi.gpts.yaml`** — схема действий, которую ты вставляешь в GPT Actions.

**GPT Actions** — механизм, через который твой Custom GPT вызывает локальный runtime.

## Часть 2. Установи Python 3.13

### Что такое Python простыми словами

Это основной интерпретатор, на котором запускается локальный агент.

Для этого проекта подтверждён именно `python3.13`.

Не надо надеяться, что “какой-то Python на Mac уже есть и этого хватит”.

### Шаг 3. Открой Terminal

Самый простой путь:

- открой `Spotlight`;
- напечатай `Terminal`;
- нажми Enter.

### Шаг 4. Проверь установленный Python

Вставь в Terminal:

```bash
python3.13 --version
```

### Что должно получиться

Ты должен увидеть что-то вроде:

```text
Python 3.13.x
```

### Если не получилось

Если Terminal пишет, что команда не найдена, значит Python 3.13 ещё не готов.

### Шаг 5. Проверь, установлен ли Homebrew

В Terminal выполни:

```bash
brew --version
```

### Что должно получиться

Если Homebrew уже установлен, ты увидишь номер версии.

### Если не получилось

Если Terminal пишет, что `brew` не найден, сначала установи Homebrew с официального сайта `brew.sh`, а потом снова проверь:

```bash
brew --version
```

### Шаг 6. Установи Python 3.13

Самый удобный путь для macOS через Homebrew:

```bash
brew install python@3.13
```

После установки полезно сразу обновить shell-сессию: закрой Terminal и открой его заново, либо открой новое окно Terminal.

### Шаг 7. Проверь Python ещё раз

Снова выполни:

```bash
python3.13 --version
```

Если версия показывается, можно идти дальше.

### Если `python3.13` всё ещё не находится

Иногда Python уже установлен, но текущая сессия Terminal ещё не увидела новую команду.

Попробуй по порядку:

1. полностью закрыть окно Terminal;
2. открыть Terminal заново;
3. снова выполнить `python3.13 --version`.

Если команда всё равно не находится, вернись к шагу с Homebrew и проверь, что установка Python 3.13 действительно завершилась без ошибки.

`ngrok` даёт внешнюю ссылку на твой локальный агент.

В Terminal выполни:

ngrok version

### Если не получилось

Если команды нет, установи `ngrok`.

Для macOS самый простой путь:

```bash
brew install ngrok/ngrok/ngrok
```

После установки снова выполни:

```bash
ngrok version
```

## Часть 4. Настрой ngrok account, authtoken и домен

### Шаг 9. Зарегистрируйся или войди в ngrok

Открой сайт `ngrok` и зайди в аккаунт.

Это шаг, который обычно нельзя полностью автоматизировать без твоего участия.

### Шаг 10. Найди authtoken

В кабинете `ngrok` открой страницу с `authtoken` и скопируй значение.

### Важно

Не публикуй этот токен в открытых местах.

### Шаг 11. Подключи authtoken на компьютере

В Terminal выполни:

```bash
ngrok config add-authtoken <ТВОЙ_ТОКЕН>
```

Вместо `<ТВОЙ_ТОКЕН>` вставь реальный токен.

### Что должно получиться

`ngrok` должен сообщить, что токен сохранён.

### Шаг 12. Создай домен в ngrok

В кабинете `ngrok` создай домен для бесплатного или подходящего плана.

### Очень важно: что именно копировать

В `.env` потом нужен только домен, без `https://`.

Правильно:

```text
my-name.ngrok-free.dev
```

Неправильно:

```text
https://my-name.ngrok-free.dev/
```

## Часть 5. Создай токен агента и файл `.env`

### Что такое AGENT_TOKEN

Это секрет, который потом GPT будет использовать для вызовов к агенту.

Он должен быть длинным и случайным.

### Шаг 13. Сгенерируй AGENT_TOKEN

В Terminal выполни:

```bash
python3 - <<'PY'
import secrets
print(secrets.token_urlsafe(48))
PY
```

Скопируй результат.

### Шаг 14. Перейди в корень проекта

В Terminal сначала зайди в папку проекта.

Пример:

```bash
cd /Users/<твоё_имя>/SecondLane
```

Если проект лежит в другой папке, подставь свой реальный путь.

### Шаг 15. Создай `.env` из шаблона

В корне проекта выполни:

```bash
cp .env.example .env
```

### Что должно получиться

После этого в корне проекта должны лежать рядом два файла:

- `.env.example`
- `.env`

### Шаг 16. Открой `.env`

Самый простой вариант через VS Code:

```bash
code .env
```

Если VS Code не подключён к команде `code`, используй один из запасных вариантов:

```bash
open -a "Visual Studio Code" .env
```

или:

```bash
open -e .env
```

Первая команда откроет файл в VS Code как приложение macOS.
Вторая команда откроет файл в TextEdit.

### Шаг 17. Заполни обязательные поля `.env`

Минимум нужно заполнить:

- `AGENT_TOKEN`
- `NGROK_DOMAIN`
- `WORKSPACE_ROOTS`

### Как заполнить `AGENT_TOKEN`

Вставь только сам токен:

```text
AGENT_TOKEN=длинная_случайная_строка
```

### Как заполнить `NGROK_DOMAIN`

Вставь только домен без `https://`:

```text
NGROK_DOMAIN=my-name.ngrok-free.dev
```

### Как заполнить `WORKSPACE_ROOTS`

Первым обязательно поставь тот корень, внутри которого реально лежат твои проекты и рабочие папки.

Если ты планируешь работать в основном с папкой `Documents`, ставь её первой.
Если ты хранишь проекты в `Projects`, ставь первой `Projects`.

Пример:

```text
WORKSPACE_ROOTS=/Users/your-name/Documents:/Users/your-name/Projects:/workspace
```

### Очень важное правило про первый путь

Первый путь в `WORKSPACE_ROOTS` должен быть тем местом, где GPT чаще всего будет искать файлы в первую очередь.

Плохой вариант:

- первым указан случайный или несуществующий путь;
- нужная рабочая папка вообще не входит в список.

Хороший вариант:

- первым стоит реальная основная рабочая папка;
- все нужные каталоги действительно существуют на Mac.

### Простое объяснение `WORKSPACE_ROOTS`

Это список корней, внутри которых GPT разрешено работать с файлами.

Если нужная папка не входит в этот список, агент не сможет с ней работать.

### Шаг 18. Сохрани `.env`

После сохранения ещё раз визуально проверь, что:

- токен не пустой;
- домен реальный;
- пути не случайные и существуют.

### Мини-проверка `.env` перед запуском

Перед запуском глазами проверь три строки:

```text
AGENT_TOKEN=...
NGROK_DOMAIN=...
WORKSPACE_ROOTS=...
```

Если хотя бы одна из них пустая, не переходи дальше.

## Часть 6. Запусти локальный агент

### Шаг 19. Найди файл запуска

В корне проекта лежит файл:

```text
Запустить Second Lane.command
```

### Шаг 20. Запусти его двойным кликом

Это штатный путь запуска для macOS-версии.

Внутри он вызывает:

```bash
python3.13 gpts_agent_control.py
```

### Что произойдёт

Проект поднимет локальный daemon, панель управления и туннель.

### Если macOS показывает предупреждение

Иногда macOS может попросить подтвердить запуск.

Разреши запуск этого файла, если ты доверяешь проекту и запускаешь его из своей локальной копии.

### Шаг 21. Дождись открытия панели

После успешного запуска должна открыться панель `Second Lane`.

### Если `.command` не запускается с первого раза

На macOS это часто связано не с проектом, а с защитой запуска.

Попробуй по порядку:

1. нажми правой кнопкой на `Запустить Second Lane.command`;
2. выбери `Open`;
3. если macOS спрашивает подтверждение, снова нажми `Open`.

Если файл всё равно не запускается, открой Terminal в корне проекта и выполни:

```bash
chmod +x "Запустить Second Lane.command"
./Запустить\ GPTS\ Agent.command
```

### Шаг 21. Дождись открытия панели

После успешного запуска должна открыться панель `Second Lane`.

### Что должно получиться

Успешный запуск выглядит так:

- панель действительно открылась, а не просто мелькнула и закрылась;
- ты видишь, что локальный агент стартовал без ошибки;
- ты видишь реальный публичный URL `ngrok`;
- нет сообщения о пустом `AGENT_TOKEN` или невалидном `NGROK_DOMAIN`.

## Часть 7. Найди публичный URL и проверь OpenAPI

### Шаг 22. Найди публичный URL

После старта панели должен появиться публичный URL вида:

```text
https://something.ngrok-free.dev
```

### Шаг 23. Проверь `openapi.gpts.yaml`

Открой `openapi.gpts.yaml` и найди первый `servers -> url`.

### Что должно получиться

Там должен быть реальный URL туннеля, а не шаблонный placeholder.

Если placeholder остался, не надо руками переписывать YAML.

Это значит, что надо разбираться с запуском панели или туннеля.

### Быстрая визуальная проверка OpenAPI

Если всё обновилось правильно, в `openapi.gpts.yaml` первая строка `servers -> url` должна выглядеть как настоящий `https://...ngrok...` адрес.

Если там всё ещё шаблон вроде `your-domain`, дальше идти нельзя.

### Стоп-проверка перед переходом к GPT

До перехода в ChatGPT у тебя уже должно быть подтверждено всё это:

- `python3.13 --version` работает;
- `ngrok version` работает;
- `.env` заполнен;
- панель открылась;
- туннель живой;
- `openapi.gpts.yaml` содержит реальный URL.

## Часть 10. Настрой аутентификацию очень подробно

### Шаг 33. Найди `AGENT_TOKEN` в `.env`

Открой `.env` и найди строку `AGENT_TOKEN=`.

### Шаг 34. Пойми главное правило

Обычно в ChatGPT UI уже выбран тип `Bearer`.

В этом случае чаще всего нужно вставить только сырое значение токена.

### Основной сценарий

Если UI уже показывает `Bearer`, вставь только сам токен, без `AGENT_TOKEN=`.

### Аварийный запасной вариант

Только если интерфейс прямо требует полный заголовок, тогда используй формат:

```text
Bearer <твой_токен>
```

### Чего делать нельзя

Нельзя вставлять вот так:

```text
AGENT_TOKEN=<твой_токен>
```

### Что должно получиться

После настройки auth в `Actions` не должно быть ощущения, что ты вставлял переменную окружения целиком. Там должен быть именно токен в нужном формате.

### Шаг 36. Проверь action в тестовом окне редактора

## Часть 13. Частые ошибки

### Ошибка: `python3.13 --version` не работает

Значит Python 3.13 не установлен или не попал в PATH.

### Ошибка: `ngrok version` не работает

Значит `ngrok` не установлен или тоже не попал в PATH.

### Ошибка: панель не открывается

Сначала проверь `.env`, потом проверь, что `python3.13` реально доступен, потом попробуй запуск из Terminal командой:

```bash
python3.13 gpts_agent_control.py
```

Если ошибка только при двойном клике по `.command`, но ручной запуск в Terminal работает, проблема почти наверняка в запуске файла macOS, а не в самом проекте.

### Ошибка: в `openapi.gpts.yaml` остался placeholder

Это не задача на ручное редактирование YAML.

Нужно чинить запуск панели или туннель.

### Ошибка: GPT не вызывает actions

Проверь четыре вещи:

1. панель не закрыта;
2. tunnel живой;
3. `openapi.gpts.yaml` содержит реальный URL;
4. в `Actions` вставлен правильный token.

## EN

`docs/MAC_FIRST_START.md` is the detailed beginner-first macOS setup guide for Second Lane.

Its practical goal is to take a user from an unprepared local folder to a working GPT with Actions.

The real setup path is:

1. put the project in a simple macOS folder
2. verify Homebrew
3. install or verify Python 3.13
4. install or verify ngrok
5. complete ngrok account, auth token, and domain steps
6. create and fill `.env`
7. launch `Запустить Second Lane.command`
8. confirm the panel and tunnel are really alive
9. confirm `openapi.gpts.yaml` now contains the real public URL
10. open the GPT builder in ChatGPT
11. paste `gpts/system_instructions.txt` into Instructions
12. upload files from `gpts/knowledge/` into Knowledge
13. paste `openapi.gpts.yaml` into Actions
14. configure Bearer auth with the real `AGENT_TOKEN`
15. run a first action call such as `getCapabilities`

Important operational details:

- if `brew` is missing, install Homebrew first and verify `brew --version`
- if `code .env` does not work, use `open -a "Visual Studio Code" .env` or `open -e .env`
- if the `.command` launcher is blocked by macOS, try right-click -> `Open`, or use Terminal with `chmod +x` and run it directly
- do not hand-edit `openapi.gpts.yaml` if the tunnel did not come up
- do not treat the setup as complete until a simple action like `getCapabilities` actually works
