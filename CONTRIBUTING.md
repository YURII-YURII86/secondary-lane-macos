## RU

Спасибо, что рассматриваешь вклад в Second Lane.

### Чем можно помочь

- сообщать о багах
- улучшать документацию
- предлагать более ясное позиционирование или onboarding
- улучшать самые ценные workflow-actions
- добавлять тесты вокруг основного GPT-сценария

### Перед началом

- предпочитай маленькие и точечные изменения
- держи публичные формулировки конкретными и честными
- не раздувай GPT-facing схему слабыми “удобными” actions
- сохраняй главное обещание проекта: работа в ChatGPT не должна останавливаться после лимитов других агентов

### Pull request'ы

Хороший pull request обычно содержит:

- чётко сформулированную проблему
- самое маленькое полезное исправление
- заметки о проверке результата
- обновлённую документацию, если изменилось поведение

### Issues

Когда открываешь issue, добавь:

- что ты пытался сделать
- что произошло вместо ожидаемого
- в каком слое была проблема: daemon, tunnel, GPT Actions или docs/setup
- точный текст ошибки или скриншоты, если они есть

## EN

Thanks for considering a contribution to Second Lane.

### Ways To Help

- report bugs
- improve documentation
- suggest clearer positioning or onboarding
- improve high-value workflow actions
- add tests around the main GPT flow

### Before You Start

- prefer small, focused changes
- keep public-facing wording concrete and honest
- avoid expanding the GPT-facing schema with weak convenience actions
- preserve the project’s core promise: keep work moving in ChatGPT after agent limits

### Pull Requests

Good pull requests usually include:

- a clear problem statement
- the smallest useful fix
- verification notes
- updated docs when behavior changes

### Issues

When opening an issue, include:

- what you were trying to do
- what happened instead
- whether this was in the daemon, tunnel, GPT Actions, or docs/setup layer
- exact error text or screenshots when possible
